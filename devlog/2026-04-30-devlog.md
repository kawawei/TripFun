## 修復登入頁離線顯示問題 [04:20]

### 問題描述
用戶反映關閉網路後，登入頁顯示「載入成員失敗」錯誤，無法選擇成員進入 App，即使已完成過登入操作也無法繞過。

### 根本原因分析
1. **`tripMembersProvider` 缺乏離線快取** — 每次進入登入頁都直接呼叫後端 API 取得成員列表，斷網則直接拋出 Exception，沒有 fallback。
2. **`main.dart` 未同步讀取已登入狀態** — `authProvider` 是 `StateNotifierProvider`，其初始 state 為 `null`，需等待非同步的 `_loadSavedUser()` 完成才能切換至主畫面，期間如果 Flutter 先渲染 `LoginPage`，就會再次觸發失敗的 API 請求。

### 解決方案

#### `auth_provider.dart` — 新增成員列表離線快取
- **成功時**: 將後端回傳的成員資料序列化 (JSON) 後，存入 `SharedPreferences` 的 `cached_trip_members` key。
- **失敗時**: 先讀取本地快取，若快取存在則直接回傳，實現離線可用；若無快取才拋出錯誤。

#### `main.dart` — 提前讀取登入狀態，避免畫面閃爍
- 在 `runApp()` 之前使用 `SharedPreferences` 同步讀取 `current_user_id` 與 `current_user_name`。
- 若存在，以 `ProviderScope.overrides` 將已知的 `UserEntity` 注入 `authProvider`，讓 App 啟動時就直接進入主畫面，跳過登入頁。
