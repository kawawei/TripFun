# 功能規格說明書 (Spec) - TripFun

> **文件說明**: 本文件定義了 TripFun App 的詳細 UI 表現、業務邏輯與資料格式，作為開發與測試的依據。

## 1. 全域 UI/UX 規範 (Global Conventions)
| 狀態類別 | 表現方式 (範例) |
| :--- | :--- |
| **Loading** | 全螢幕磨砂玻璃效果 + 飛機飛行動畫 Spinner。 |
| **Empty State** | 顯示「準備好開始新的冒險了嗎？」插圖 + 「建立行程」按鈕。 |
| **幣值顯示** | 記帳模組依據設定幣別顯示（如 TWD 1,200, JPY 5,000），統一千分位。 |
| **時間格式** | 統一使用 24 小時制 (HH:mm)，如 08:30, 22:15。 |

---

## 2. 詳細功能規格 (Detailed Specs)

### 2.1 模組：行程管理 (Trip Management)
#### 2.1.1 頁面：行程列表 (Trip List)
*   **功能描述**: 顯示所有已建立的旅程（進行中、即將到來、已結束）。
*   **UI 元素**:
    *   **行程卡片**: 顯示行程名稱、日期區間、目的地國家旗幟。
    *   **新增按鈕**: 懸浮按鈕 (FAB)，點擊開啟「建立新行程」彈窗。
*   **邏輯**:
    *   按日期降序排列。
    *   提供「離線可用」標籤，標註已完成資料預載的行程。

#### 2.1.2 頁面：行程詳情 (Daily Timeline)
*   **功能描述**: 分頁顯示每日行程，精確到分鐘。
*   **UI 元素**:
    *   **日期切換欄**: 橫向捲動的日期條。
    *   **時間軸**: 垂直線條連接各個活動點。
    *   **活動卡片**:
        *   時間 (HH:mm)。
        *   標題 (目的地/活動名稱)。
        *   地圖導航圖示：點擊開啟 Google Maps 並定位至目標座標。
*   **邏輯**:
    *   活動衝突檢查：若兩個活動時間重疊，時間標籤顯示為紅色提醒。

### 2.2 模組：工具箱 (Travel Tools)
#### 2.2.1 離線翻譯 (Offline Translation)
*   **UI 元素**:
    *   **語系選擇**: 來源語系（預設自動辨識/繁中）與目標語系（選取）。
    *   **輸入框**: 文字輸入區。
    *   **結果區**: 顯示翻譯文字，並提供一鍵「全螢幕放大」功能，方便展示給當地人看。
*   **邏輯**:
    *   **離線支援**: 檢查本地有無該語系包，若無則提示下載。
    *   **歷史紀錄**: 自動保存最近 20 筆翻譯紀錄。

#### 2.2.2 旅途記帳 (Individual Accounting)
*   **UI 元素**:
    *   **金額輸入**: 支援簡易加減乘除小鍵盤。
    *   **分類選擇**: 交通、食物、住宿、購物、其他。
    *   **行程過濾**: 自動關聯至當前行程。
*   **邏輯**:
    *   匯率自動帶入：若有網路，依據當日匯率預算台幣價值；若無網路，使用最後一次更新的匯率。

#### 2.2.3 跨時區鏡 (Timezone Mirror)
*   **功能描述**: 解決跨國旅遊的生理時鐘與行程安排偏差。
*   **UI 元素**:
    *   **目前位置鐘**: 獲取 GPS 權限後顯示所在地時間。
    *   **目的地鐘**: 可添加 1-3 個目的地名稱與對應時間。
*   **邏輯**:
    *   **溫差提示**: 同步顯示各時間點下的氣溫（需聯動天氣 API）。

### 2.3 住宿與交通資訊存儲 (Booking Vault)
*   **欄位定義**:
    *   **班機**: 航空公司、班次、起飛/抵達時間、航廈、登機口、航空公司官網連結、PDF/圖片附件。
    *   **住宿**: 飯店名稱、地址（含當地語言複製按鈕）、入住/退房時間、訂房代號、飯店連絡電話。

### 2.4 模組：家屬協作與共享 (Family Collaboration)
*   **功能描述**: 允許多人同步查看與編輯行程。
*   **邀請方式**:
    *   **安全連結**: 生成帶有唯一加密 Token (UUID) 的 URL。
    *   **QR Code**: 掃描包含該連結的動態 QR Code。
*   **角色權限**:
    *   **Owner (建立者)**: 擁有最高權限（含刪除行程）。
    *   **Manager (管理者)**: 可編輯行程、邀請他人。
    *   **Viewer (檢視者)**: 僅供查看，不可更動資料。
*   **邏輯**:
    *   實時同步：偵測到資料庫變更時顯示同步提示（如：「媽媽剛更新了飯店資訊」）。

## 3. 欄位屬性表 (Data Schema Map)

| 模組 | 欄位名稱 | 類型 | 限制 | 備註 |
| :--- | :--- | :--- | :--- | :--- |
| 行程 | `trip_name` | String | 1-50 chars | 必填 |
| 活動 | `start_time` | DateTime | HH:mm | 必填 |
| 記帳 | `amount`| Decimal | 10.2 | 必填 |
| 翻譯 | `text` | Text | 1000 chars | |

---

## 4. 權限需求
1.  **定位權限 (GPS)**：用於時區自動跳轉與翻譯語系初判。
2.  **相機/相簿權限**：用於存儲機票與飯店訂單截圖。
3.  **背景下載權限**：用於在背景下載離線翻譯包與地圖數據。
